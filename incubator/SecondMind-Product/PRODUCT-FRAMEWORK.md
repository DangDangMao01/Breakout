# 个人 AI 知识中枢 - 产品框架文档

> 版本: v0.1 (初稿)  
> 日期: 2025-12-25  
> 状态: 构思阶段

---

## 一、产品愿景

**一句话定位：** 跨平台、跨 AI 的个人知识聚合中枢

**核心理念：** 不是笔记本，是你的**第二大脑**

- 记住你学过的一切
- 理解你正在思考的问题
- 主动帮你发现知识盲区
- 连接你所有的 AI 对话

---

## 二、目标用户

| 用户类型 | 痛点 | 需求 |
|---------|------|------|
| 程序员 | AI 对话分散，知识难复用 | 跨项目知识积累 |
| 学习者 | 学过就忘，缺乏系统整理 | 知识图谱、学习路径 |
| 创作者 | 灵感转瞬即逝 | 随时记录、智能关联 |
| 知识工作者 | 信息过载，难以检索 | 语义搜索、智能推荐 |

---

## 三、核心价值主张

### 3.1 与现有产品的差异化

| 维度 | 有道云笔记/Notion | ChatGPT/Claude | **本产品** |
|------|------------------|----------------|-----------|
| 定位 | 通用笔记 | AI 对话 | **AI 知识中枢** |
| 数据归属 | 平台存储 | 平台存储 | **用户本地优先** |
| AI 集成 | 辅助功能 | 核心但分散 | **跨 AI 聚合** |
| 知识持久化 | 手动整理 | 对话历史分散 | **自动归纳** |
| 离线能力 | 部分支持 | 必须联网 | **完全离线可用** |

### 3.2 四大核心价值

1. **知识归属权** - 你的知识属于你，不是平台的训练数据
2. **跨 AI 聚合** - 无论用哪个 AI，知识都汇聚到一处
3. **本地 AI 友好** - 支持 Ollama 等本地模型，完全离线使用
4. **长期记忆** - AI 会忘，但你的知识库不会

## 五、核心功能

### 5.1 AI 中枢能力

| 能力 | 描述 | 实现方式 |
|------|------|---------|
| 🧠 智能总结 | 长对话/文章一键提炼要点 | LLM 摘要 |
| ❓ 智能提问 | 根据知识库内容，AI 主动提出深入问题 | 知识图谱分析 |
| 🔗 智能归纳 | 自动发现知识间的关联，形成知识网络 | 向量相似度 + 标签聚类 |
| 📚 智能整理 | 未知领域知识点自动分类、建立学习路径 | 领域分类模型 |
| 🔍 智能检索 | 语义搜索，不只是关键词匹配 | RAG + Embedding |

### 5.2 用户体验设计

**"想用就用"的关键：**

| 场景 | 用户行为 | 产品响应 |
|------|---------|---------|
| 突然有想法 | 快捷键 `Ctrl+Alt+Q` | 秒级记录，不打断工作流 |
| 遇到问题 | 描述问题 | AI 自动搜索知识库，给出相关内容 |
| 学习新知识 | 粘贴/导入内容 | AI 自动总结、打标签、归类 |
| 回顾知识 | 打开知识库 | AI 生成知识图谱、关联推荐 |
| 跨领域探索 | 提问未知领域 | AI 整理知识点，形成学习路径 |

**体验原则：**
- 🚀 **极速调用** - 全局快捷键，任何场景 1 秒唤起
- 🧠 **AI 主动服务** - 不只是被动存储，主动分析、提醒、推荐
- 🔗 **无缝集成** - 多端全覆盖，数据实时同步
- 📊 **可视化** - 知识图谱、学习进度、知识盲区一目了然

### 5.3 AI 对接能力

| AI 类型 | 对接方式 | 优先级 |
|---------|---------|--------|
| Kiro | 插件内置 | P0 ✅ |
| Ollama (本地) | REST API | P1 |
| ChatGPT | API / 浏览器插件 | P1 |
| Claude | API / 浏览器插件 | P1 |
| 其他本地模型 | OpenAI 兼容 API | P2 |
