# Kiro KB 产品愿景：为个人机器人时代做准备

## 核心愿景

**Slogan**: 今天的知识管理，明天的机器人大脑

**长期目标**: 构建一个可迁移的个人知识资产系统，为未来的个人机器人提供"思维模式"和"记忆基础"。

## 战略定位

### 现在 vs 未来

```
┌─────────────────────────────────────────────────────────┐
│  现在（2026）              未来（5-10 年后）            │
├─────────────────────────────────────────────────────────┤
│  个人知识库          →     个人机器人的"大脑"          │
│  Markdown + AI       →     机器人 + 知识库 + AI        │
│  工作效率工具        →     生活伴侣 + 工作助手         │
└─────────────────────────────────────────────────────────┘
```

### 核心逻辑

1. **未来家用机器人会普及**
   - 就像现在人人有智能手机
   - 未来人人会有个人机器人

2. **出厂机器人 = 通用能力**
   - 像 iPhone 出厂设置
   - 有基础功能，但不了解你

3. **个人机器人 = 通用能力 + 你的知识库**
   - 导入你的知识库
   - 机器人"认识"你
   - 理解你的工作习惯、生活偏好、思维模式

4. **你的知识库 = 你的数字分身**
   - 工作习惯和技能
   - 生活偏好和经验
   - 学习内容和思考

### 类比

**就像现在**:
- 买新手机 → 登录 iCloud → 恢复所有数据
- 换电脑 → 同步 Git → 恢复所有代码

**未来会是**:
- 买机器人 → 导入知识库 → 机器人"认识"你
- 换机器人 → 迁移知识库 → 无缝切换

## 产品定位

### 不是什么

- ❌ 又一个笔记工具（Notion/Obsidian）
- ❌ 对话式 AI（ChatGPT）
- ❌ 企业监控工具
- ❌ 云端服务（依赖厂商）

### 是什么

- ✅ **可迁移的个人知识资产系统**
- ✅ **为未来机器人准备的"记忆基础"**
- ✅ **本地化、隐私优先的 AI 助手**
- ✅ **跨设备、跨平台的知识管理**

### 核心价值主张

**对用户**:
1. 今天：提升工作效率，管理知识
2. 明天：为个人机器人提供"思维模式"
3. 永远：知识归你所有，可迁移、可控制

**对市场**:
1. 差异化：不是笔记工具，是"数字分身"
2. 前瞻性：为机器人时代做准备
3. 伦理性：尊重个人数据所有权

## 三阶段实施路线

### Phase 1: 工作知识管理（6-12 个月）✅ **当前重点**

**目标**: 成为"开发者的第二大脑"

**核心功能**:
1. Ollama 本地 AI 集成
2. 工作模式追踪（文件访问、搜索、Git 提交、编辑时间）
3. 日报/周报自动生成
4. 个人技能树管理（profile.yaml）
5. 跨项目知识复用
6. 个人成长仪表盘

**用户画像**:
- 开发者、设计师、技术写作者
- 重视隐私和数据所有权
- 希望提升工作效率
- 愿意为个人工具付费

**验证指标**:
- 100+ 活跃用户
- 用户平均使用 3 个月以上
- 用户愿意付费（$5-10/月）
- NPS > 50

**技术栈**:
- VSCode Extension (TypeScript)
- Ollama (本地 AI)
- Markdown + Git (知识存储)
- fast-check (属性测试)

---

### Phase 2: 知识消耗（12-24 个月）📚 **扩展方向**

**目标**: 扩展到"外部知识内化"

**核心功能**:
1. **PDF 解析 + AI 摘要**
   - 用户拖入 PDF 论文
   - AI 生成通俗易懂的摘要
   - 提取关键概念 → 建立知识卡片
   - 标注难点 → 生成学习路径

2. **视频转文字 + 知识提取**
   - 视频 URL → 提取字幕/语音转文字
   - AI 分析内容 → 生成结构化笔记
   - 时间戳标注 → 方便回看重点

3. **渐进式学习系统**
   - 第一遍：AI 生成"小白版"解释
   - 第二遍：逐步深入，补充细节
   - 第三遍：关联已有知识，形成网络

4. **论文阅读助手**
   - 识别论文结构（摘要、方法、结论）
   - 提取核心贡献
   - 关联相关论文
   - 生成阅读笔记

**使用场景**:
- 普通开发者想学习尖端知识（AI、量子计算、前沿算法）
- 下载了很多 PDF 论文 → 看不懂，放着吃灰
- 收藏了很多技术视频 → 没时间看，或看了记不住
- 想学习前沿知识 → 缺少系统化的消化路径

**技术准备**:
- 多模态 AI 模型（LLaVA、Whisper）
- 文档解析库（PDF.js、Tesseract OCR）
- 视频处理（FFmpeg、字幕提取）
- 知识图谱（关联概念）

---

### Phase 3: 生活知识 + 机器人准备（24+ 个月）🤖 **长期愿景**

**目标**: 成为"全生活知识助手"，为机器人集成做准备

**核心功能**:
1. **生活技能学习**
   - 做菜、健身、园艺、手工
   - 多模态输入（摄像头、图片）
   - AI 识别步骤和关键点
   - 生成个人化教程

2. **实时视觉辅助**
   - 摄像头实时拍摄
   - AI 识别：食材、工具、动作
   - 自动生成结构化笔记
   - 下次做菜时，AI 提醒关键步骤

3. **机器人接口**
   - 标准化知识库格式
   - 机器人 API 接口
   - 知识库导入/导出
   - 跨设备同步

4. **移动端支持**
   - iOS/Android App
   - 随时随地记录知识
   - 语音输入 + AI 转文字
   - 拍照 + AI 识别

**使用场景**:
- 学做菜：摄像头拍摄 → AI 识别步骤 → 生成菜谱
- 学健身：视频分析 → 动作识别 → 生成训练计划
- 学手工：图片记录 → 步骤提取 → 生成教程
- 未来：导入到个人机器人 → 机器人学会你的技能

**技术准备**:
- 视觉识别 AI（物体识别、动作识别）
- 实时视频处理
- 移动端开发（React Native / Flutter）
- 机器人 API 标准（ROS / 自定义协议）

## 核心设计原则

### 1. 知识外化存储 📦

**原则**: AI 是工具，知识是资产

**实现**:
- 所有知识 → Markdown 文件
- 所有配置 → YAML 文件
- 所有数据 → Git 同步
- AI 只是分析工具，可随时替换

**价值**:
- 换电脑 → 知识不丢失
- 换工作 → 经验可带走
- 换 AI → 知识仍然有效
- 换机器人 → 知识可迁移

### 2. 隐私优先 🔒

**原则**: 本地处理，用户控制

**实现**:
- 所有 AI 处理 → 本地 Ollama
- 所有数据存储 → 本地文件系统
- 所有同步 → Git（用户控制）
- 无云端依赖 → 无数据泄露风险

**价值**:
- 数据归你所有
- 隐私完全保护
- 不依赖厂商
- 永久可用

### 3. 可迁移性 🚀

**原则**: 跨设备、跨平台、跨时间

**实现**:
- 知识库 → Git 仓库
- 配置文件 → 纯文本
- AI 模型 → 可替换
- 接口标准 → 开放协议

**价值**:
- 换设备 → 无缝迁移
- 换平台 → 完全兼容
- 换 AI → 知识保留
- 换机器人 → 直接导入

### 4. 渐进式演进 📈

**原则**: 先做好基础，再扩展功能

**实现**:
- Phase 1 → 工作知识管理
- Phase 2 → 知识消耗
- Phase 3 → 生活知识 + 机器人

**价值**:
- 快速验证产品价值
- 降低技术风险
- 积累用户基础
- 保持产品聚焦

## 风险与挑战

### 技术风险

1. **多模态 AI 性能**
   - 本地运行视觉模型 → 硬件要求高
   - 缓解：提供云端选项（用户控制）

2. **跨平台兼容**
   - Windows/Mac/Linux 差异
   - 缓解：使用跨平台技术栈

3. **数据同步冲突**
   - Git 合并冲突
   - 缓解：智能冲突解决

### 市场风险

1. **用户教育成本**
   - "为机器人做准备"概念新颖
   - 缓解：先强调工作效率，再引入愿景

2. **竞争压力**
   - Notion/Obsidian 功能丰富
   - 缓解：差异化定位（数字分身）

3. **付费意愿**
   - 免费工具众多
   - 缓解：提供独特价值（本地 AI + 隐私）

### 伦理风险

1. **数据滥用**
   - 如果做企业版 → 可能监控员工
   - 缓解：**坚持做个人工具**

2. **AI 偏见**
   - AI 分析可能有偏见
   - 缓解：透明化 AI 决策，用户可审查
