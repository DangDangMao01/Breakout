## 🎯 Clawdbot 定位与特性

### 核心定位

**24 小时全天候运行在本地设备上的个人 AI 助理**
- 拥有操作系统最高权限
- 能替用户干活的自主代理 Agent
- 不是"清谈客"，而是"数字合伙人"

### 架构特点

**本地优先架构**：
- 所有设置偏好和对话记忆以 **Markdown 文档**形式存储
- 存储位置：本地硬盘的 `clawd/` 目录
- 数据完全由用户控制，不上传云端
- 隐私性极高，类似 Obsidian 的设计

**语义检索长时记忆**：
- 真正的长期记忆（不是会话级）
- 能记住用户之前的偏好（如饮食偏好）
- 在后续任务中给出个性化建议
- 24/7 上下文保留系统

## 🖥️ 运行环境与硬件选择

### 为什么 Mac mini 是最佳宿主机？

**1. 生态需求**：
- 调用 iMessage 原生权限
- 访问 iCloud 文件
- 管理日历
- 需要 macOS 设备 24 小时在线

**2. 性价比与能效**：
- M 系列芯片功耗低
- 散热好
- 适合运行本地大模型
- 适合 24/7 代理服务

**3. 极客堆叠现象**：
- 多台 Mac mini 堆叠
- 打造私人 AI 算力中心
- 形成分布式 AI 集群

### 其他运行环境

- ✅ Linux VPS
- ✅ Raspberry Pi
- ✅ 旧笔记本
- ✅ Windows（需要配置）

## ⚠️ 风险与挑战

### 安全隐患

**完整的 Shell 权限**：
- 指令不当可能导致误删文件
- 可能泄露敏感数据
- 提示词注入攻击风险

### 成本问题

**API 调用成本较高**：
- 24/7 运行消耗大量 API 调用
- 需要付费 API（Claude、GPT-4）
- 月度成本可能较高

### 技术门槛

**需要一定技术能力**：
- 配置服务器
- 管理权限
- 调试问题

## 🔍 与 Kiro KB 插件的关联

### 核心相似点

| 维度 | Clawdbot | Kiro KB 插件 |
|------|----------|--------------|
| **存储方式** | Markdown 文档 | Markdown 文档 |
| **存储位置** | 本地 `clawd/` | 本地 `knowledge-base/` |
| **隐私优先** | ✅ 完全本地 | ✅ 完全本地 |
| **记忆机制** | 长期记忆 | 对话记录 |
| **数据控制** | 用户完全控制 | 用户完全控制 |
| **版本控制** | 可用 Git | 可用 Git |

### 核心差异点

| 维度 | Clawdbot | Kiro KB 插件 |
|------|----------|--------------|
| **定位** | 自主代理 Agent | 知识管理工具 |
| **权限** | 系统级权限 | IDE 内权限 |
| **主动性** | 主动执行任务 | 被动保存知识 |
| **执行能力** | 可执行系统命令 | 不执行命令 |
| **运行方式** | 24/7 后台运行 | IDE 插件 |
| **交互方式** | 消息应用 | IDE 界面 |

### 可借鉴的设计

1. **Markdown 存储**
   - Clawdbot 用 Markdown 存储记忆
   - Kiro KB 也用 Markdown 存储知识
   - 两者可以共享数据格式

2. **语义检索**
   - Clawdbot 有语义检索长时记忆
   - Kiro KB 可以实现类似功能
   - 基于向量数据库

3. **自我进化**
   - Clawdbot 可以自写代码自装插件
   - Kiro KB 可以实现技能自动生成
   - 基于 AI 驱动

4. **主动性**
   - Clawdbot 可以主动提醒
   - Kiro KB 可以实现主动推荐
   - 基于定时任务
