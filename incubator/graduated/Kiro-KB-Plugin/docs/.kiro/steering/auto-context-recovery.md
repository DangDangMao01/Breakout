# 自动上下文恢复

每次新会话开始时，如果用户说"继续开发"或类似意图，必须先读取以下文件：

1. `SESSION-STATE.md` - 项目整体状态和当前进度
2. `DEVELOPMENT-ROADMAP.md` - 开发计划和任务列表
3. `kiro-knowledge-base/CHANGELOG.md` - 版本变更历史

## 读取后确认

向用户确认以下信息：
- 当前版本号
- 最近完成的工作
- 下一步计划
- 已知问题（如有）

## 跨设备开发原则

1. **先读后写** - 新设备上，先读取所有上下文，确认理解正确后再开始开发
2. **小步提交** - 每完成一个小功能就更新文档
3. **打包前验证** - 每次打包前检查版本号和功能完整性
4. **文档同步** - 代码变更必须同步更新 SESSION-STATE.md

## 重要提醒

- 不要假设上次的上下文还在
- 不要跳过读取步骤
- 如果用户说"继续"，必须先确认上下文正确
