# Kiro KB 产品核心理念

> 这是产品的"灵魂"，所有开发决策都应该符合这些原则

## 🎯 产品定位

**一句话**：让 AI 越来越懂你的个人记忆层

**核心价值**：
- 不是通用知识库（那不如下载本地 AI）
- 而是"你的"知识库，让 AI 懂"你"
- AI 是工具，但工具要懂主人

**与通用 AI 的区别**：
| 通用 AI | Kiro KB + AI |
|---------|--------------|
| 每次对话从零开始 | 记得你之前说过什么 |
| 不知道你会什么 | 知道你擅长动画特效 |
| 解释你已经懂的 | 跳过基础，直接进阶 |
| 给通用建议 | 给适合你的建议 |

## 🧭 设计原则

1. **隐私优先** - 本地存储，用户完全控制数据
2. **可迁移性** - Markdown + Git，不依赖特定平台
3. **AI 辅助而非依赖** - AI 可随时替换，知识永久保留
4. **简单直接** - 减少层级，快速访问

## 📍 开发阶段

**Phase 1: 工作知识管理**（当前）
- 保存 Kiro 对话
- 知识分类和搜索
- 跨设备同步

**Phase 2: 知识消耗**（下一步）
- 手机端信息同步（Telegram Bot）
- PDF/网页内容提取
- 外部知识内化

**Phase 3: 本地 AI 记忆**（未来）
- Ollama 集成
- 智能问答
- 工作报告生成

## ⚠️ 不做什么

- 不做云端存储（隐私风险）
- 不做社交分享（专注个人）
- 不做复杂的知识图谱可视化（保持简单）
- 不预测市场/投资（AI 能力边界）

## 💡 用户痛点

1. **AI 不懂我** - 每次都要重新解释背景
2. **知识碎片化** - 学了很多，但不成体系
3. **跨设备丢失** - 换设备后上下文断裂
4. **平台锁定** - 知识困在某个 App 里

## 🧠 让 AI 越来越懂你的机制

### 1. 用户画像（自动积累）
- **擅长领域** - 你教 AI 的内容 = 你会的
- **知识边界** - 你问 AI 的内容 = 你不会的
- **偏好习惯** - 你的选择 = 你的风格

### 2. 知识分层
- **你已经会的** → AI 不解释基础，直接进阶
- **你学过但忘了** → AI 提醒关键点
- **你完全不懂的** → AI 从头解释

### 3. 增量学习
- 只保存对你有增量价值的内容
- 你问的 = 你不知道的 = 必须保存
- AI 给出更好方案 = 超出你已知 = 必须保存
- 你教 AI 的 = 你已经会的 = 不保存（但记录到画像）

## 🔧 技术栈

- **存储**: Markdown 文件 + Git
- **IDE**: Kiro/VSCode/Trae 插件
- **本地 AI**: Ollama (可选)
- **同步**: Git / 云盘
