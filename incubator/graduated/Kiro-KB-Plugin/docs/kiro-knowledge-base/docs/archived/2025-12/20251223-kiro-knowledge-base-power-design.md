# Kiro 知识库 Power 设计构思

## 背景

用户希望将现有的知识库脚本整合成一个可安装的 Kiro 插件/Power，实现：
- 一键安装配置
- 自动关联所有 Kiro 打开的项目
- 智能统计分类问题解决记录
- 自动同步到中央知识库

## 两种实现路径

### 方案 A：Kiro Power（长期目标）
- 打包成可安装的功能模块
- 包含 MCP Server 提供工具
- 有 Steering 文件引导使用
- 可发布给其他用户

### 方案 B：一键安装脚本包（短期可行）
- 运行一次完成所有配置
- 自动创建全局 Steering 规则
- 自动注册 Hook 模板
- 配置中央知识库路径

## 核心功能设计

```
📦 Kiro Knowledge Base Power
├── 安装时
│   ├── 询问中央知识库位置（或创建新的）
│   ├── 配置全局 Steering 规则
│   └── 设置自动同步机制
│
├── 每个项目打开时
│   ├── 检测是否有历史对话记录
│   ├── 提示是否启用自动保存
│   └── 显示未完成任务（如有）
│
└── 自动运行
    ├── 对话结束时智能保存
    ├── 自动添加标签分类
    └── 定期同步到中央库
```

## 建议路线

1. 先做方案 B 验证流程
2. 成熟后打包成 Kiro Power

## 下一步

等待用户确认方向后开始设计安装脚本包结构
