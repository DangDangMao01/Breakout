# Skills 系统详解 - 从自动化到 AI 能力模块

**日期**: 2026-01-28  
**来源**: Claude Skills (Anthropic, 2025) + Trae Skills + 用户疑问

---

## 🤔 什么是 Skills？

### 简单理解

**Skills = AI 的"专业技能包"**

就像：
- 人类学习"开车"这个技能
- 学会后，遇到车就知道怎么开
- 不需要每次都从头学

AI 的 Skills：
- 预先定义好的"工作流程"
- AI 遇到相关任务时自动加载
- 不需要每次都重新解释

---

## 📚 Skills 的演变历史

### 阶段 1: 传统自动化（早期）

**博主说的"早期自动化"指的是这个**

```
传统自动化工具：
├── Zapier（连接应用）
├── IFTTT（If This Then That）
├── n8n（工作流自动化）
└── RPA（机器人流程自动化）

特点：
- 固定流程
- 无法适应变化
- 需要手动配置每一步
```

**例子**：
```
当收到邮件（触发）
→ 提取附件（步骤1）
→ 保存到 Google Drive（步骤2）
→ 发送通知（步骤3）
```

**问题**：
- ❌ 流程固定，无法灵活调整
- ❌ 遇到新情况就失败
- ❌ 需要大量手动配置

---

### 阶段 2: AI Agent 工作流（2023-2024）

```
AI Agent 自动化：
├── LangChain（AI 工作流框架）
├── AutoGPT（自主 AI）
├── BabyAGI（任务分解）
└── n8n + AI（混合方案）

特点：
- AI 可以决策
- 可以适应变化
- 但每次都要"思考"
```

**例子**：
```
用户：帮我整理邮件

AI Agent：
1. 读取邮件列表
2. 分析每封邮件内容
3. 决定分类规则
4. 执行分类
5. 生成报告
```

**问题**：
- ❌ 每次都要重新"思考"
- ❌ 消耗大量 Token
- ❌ 速度慢
- ❌ 成本高

---

### 阶段 3: Skills 系统（2025-现在）

**Anthropic 在 2025 年 10 月发布 Claude Skills**

```
Skills = 可复用的 AI 能力模块

特点：
- 预先定义好流程
- AI 自动识别何时使用
- 按需加载（Progressive Disclosure）
- 低成本（30-50 tokens）
```

**例子**：
```
# email-organizer.skill/SKILL.md

## 技能名称
邮件整理专家

## 何时使用
- 用户提到"整理邮件"
- 用户提到"邮件分类"
- 检测到大量未读邮件

## 工作流程
1. 读取邮件列表
2. 按以下规则分类：
   - 工作邮件 → Work 文件夹
   - 个人邮件 → Personal 文件夹
   - 营销邮件 → Promotions 文件夹
3. 标记重要邮件
4. 生成摘要报告

## 参考资料
- 用户的邮件偏好设置
- 历史分类记录
```

**优势**：
- ✅ 不需要每次重新思考
- ✅ 只消耗 30-50 tokens（元数据）
- ✅ 速度快
- ✅ 成本低
- ✅ 可复用

## 🔄 Skills 的工作原理

### Progressive Disclosure（渐进式披露）

**这是 Skills 的核心机制！**

```
阶段 1: 元数据加载（30-50 tokens）
├── 技能名称
├── 简短描述
├── 触发条件
└── 标签

AI 判断：这个 Skill 相关吗？
├── 相关 → 进入阶段 2
└── 不相关 → 跳过

阶段 2: 完整加载（数百到数千 tokens）
├── 详细工作流程
├── 参考资料
├── 示例数据
└── 脚本代码
```

**优势**：
- 不相关的 Skill 只消耗 30-50 tokens
- 相关的 Skill 才完整加载
- 大幅降低成本和延迟

---

### 自动路由

```
用户：帮我整理邮件

AI 内部流程：
1. 扫描所有 Skills 的元数据
2. 发现 "email-organizer" 相关
3. 加载完整的 SKILL.md
4. 按照定义的流程执行
5. 返回结果
```

**类比**：
```
就像你的大脑：
- 听到"开车" → 自动激活"开车技能"
- 不需要每次都重新学习
- 直接执行已知流程
```

| **可复用** | 低 | 低 | 高 |

**结论**：Skills 是最佳平衡点

## 💡 我们的 Skills 设计

```
不是手动创建 Skills
而是从知识库自动生成 Skills

流程：
1. 用户使用知识库
2. 系统检测重复模式
3. 自动生成 Skill
4. 用户确认和优化
5. Skill 可复用
```

---

### 自动生成示例

**场景**：用户多次问"如何调试 Cocos 脚本"

```
系统检测到：
- 3 次相似问题
- 解决方案相似度 > 80%
- 可以标准化

自动生成：
cocos-debugger.skill/SKILL.md

内容：
- 从 3 次对话中提取共同点
- 总结最佳实践
- 生成标准流程
- 添加示例代码
```

**下次遇到类似问题**：
- AI 自动加载这个 Skill
- 直接给出标准答案
- 不需要重新思考
