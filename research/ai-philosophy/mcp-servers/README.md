## 💡 接入方案（根据输出方式）

### 方案 A: 用户手动复制（最简单，立即可用）

**工作流**:
```
1. 用户在 IMA 中总结 PDF/网页/视频
2. IMA 显示总结结果
3. 用户复制总结
4. 用户粘贴给 Kiro："这是 IMA 的总结：[粘贴内容]"
5. Kiro 读取并分析
```

**优点**:
- ✅ 无需开发，立即可用
- ✅ 用户完全控制

**缺点**:
- ❌ 需要手动复制粘贴

### 方案 B: 剪贴板监控（自动化）

**工作流**:
```
1. 用户在 IMA 中总结 PDF/网页/视频
2. IMA 自动复制总结到剪贴板
3. Kiro 监控剪贴板变化
4. Kiro 自动读取并分析
5. Kiro 主动提示："我发现 IMA 的总结，是否需要分析？"
```

**优点**:
- ✅ 自动化，无需手动复制
- ✅ 用户体验好

**缺点**:
- ❌ 需要开发剪贴板监控
- ❌ 可能误触发（其他复制操作）

**实现方式**:
- 创建 Python 脚本监控剪贴板
- 检测到 IMA 特征（例如特定格式）时触发
- 通知 Kiro 读取

### 方案 C: 文件监控（自动化）

**工作流**:
```
1. 用户在 IMA 中总结 PDF/网页/视频
2. IMA 保存总结到文件（例如 `C:\Users\xxx\IMA\summaries\xxx.txt`）
3. Kiro 监控文件夹变化
4. Kiro 自动读取新文件
5. Kiro 主动提示："我发现 IMA 的总结，是否需要分析？"
```

**优点**:
- ✅ 自动化，无需手动复制
- ✅ 不会误触发
- ✅ 可以保留历史记录

**缺点**:
- ❌ 需要开发文件监控
- ❌ 需要知道 IMA 的保存路径

**实现方式**:
- 创建 Python 脚本监控文件夹
- 检测到新文件时触发
- 通知 Kiro 读取

### 方案 D: Hook 集成（最优雅）

**工作流**:
```
1. 用户在 IMA 中总结 PDF/网页/视频
2. 用户对 Kiro 说："读取 IMA 的总结"
3. Kiro 自动读取剪贴板或文件
4. Kiro 分析并回答
```

**优点**:
- ✅ 用户控制触发时机
- ✅ 不会误触发
- ✅ 可以结合剪贴板或文件监控

**缺点**:
- ❌ 需要用户主动触发

**实现方式**:
- 创建 Kiro Hook："读取 IMA 的总结"
- Hook 触发时读取剪贴板或文件
- 返回给 Kiro 分析
