## 💡 关键技术要点

### 角色一致性技术

#### 云端服务
- **Runway Gen-4 References**: 上传参考图，使用@符号引用
- **Kling 2.0 Elements**: 1-4张参考图，保持角色外观
- **Pika Character Reference**: 角色参考功能

#### ComfyUI本地
- **IPAdapter**: 注入角色特征，权重0.7-0.9
- **LoRA训练**: 20-50张图训练角色LoRA
- **固定Seed**: 所有动画使用连续seed

### 序列帧提取

#### 工具选择
- **Cocos工具**: 视频转序列帧生成器（推荐）
- **FFmpeg**: 命令行提取
- **Python脚本**: 批量处理

#### 处理流程
```
视频 → 提取帧 → 去背景 → 统一尺寸 → 游戏引擎
```

### Spine转换

#### 方法A：使用Cocos工具
```
序列帧长图 → Spine序列帧生成器 → Spine文件
```

#### 方法B：直接使用序列帧
```
PNG序列 → Unity/Cocos/Unreal → Sprite Animation
```

## ⚠️ 注意事项

### AI序列帧方案的限制

**优势**：
✅ 效率极高（2-3天完成）
✅ 成本可控
✅ 无需Spine软件
✅ 批量生产能力强

**劣势**：
⚠️ 不是真正的骨骼动画
⚠️ 包体较大（每个动画都是完整图片序列）
⚠️ 无法动态换装/换色
⚠️ 不同动作需要重新生成
⚠️ AI有随机性

### 适用场景

**✅ 适合**：
- 大量小怪/NPC动画
- 简单角色动画
- 快速原型验证
- 预算/时间有限
- 独立开发者

**❌ 不适合**：
- 主角复杂动画（建议手工）
- 需要换装系统
- 需要动态混合动画
- 对包体大小极度敏感
