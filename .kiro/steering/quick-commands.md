# 快速指令库

**用途**: 工作时间使用，最小化 AI 协作成本

**原则**: 每次协作 < 15 分钟

---

## 📋 插件开发

### 代码生成
```
生成 [功能名称] 的代码，参考 [文件路径/文档]，重点关注 [关键点]。
只生成核心代码，不要解释。
```

### Bug 修复
```
修复这个错误：[错误信息]
只改必要的代码，不要重构。
```

### 代码审查（快速）
```
检查这段代码的 [错误类型]（如：语法错误、类型错误）。
不要全面审查，只关注明显问题。
```

### 功能测试
```
测试 [功能名称] 是否正常工作。
如果有问题，直接告诉我怎么修复。
```

## 🎮 游戏开发

### 需求讨论（深度模式，非工作时间）
```
我们来讨论 [游戏名称] 的设计：
- 核心玩法
- 技术选型
- 架构设计
- 关键技术点

讨论后生成详细需求文档。
```

### 代码生成（Trae）
```
根据需求文档生成 [游戏名称] 的代码。
要求：
- 代码结构清晰
- 组件化设计
- 完整的注释
- 符合 [引擎] 最佳实践
```

### 代码审查（Kiro）
```
审查 Trae 生成的代码，重点检查：
- 游戏逻辑是否正确
- 性能优化点
- 潜在 Bug
- 代码规范

生成详细的审查报告。
```

## 🚀 工作时间规则

### 允许做的（< 15 分钟）
- ✅ 插件 Bug 快速修复
- ✅ 简单功能代码生成
- ✅ 快速代码审查
- ✅ 文档快速整理

### 不允许做的（留到晚上/周末）
- ❌ 深度架构讨论
- ❌ 复杂功能开发
- ❌ 游戏开发
- ❌ 全面代码重构

### 判断标准
**如果一个任务预计超过 15 分钟 → 留到晚上/周末**

## 💡 使用技巧

### 技巧 1: 批量处理
不要频繁切换，把类似任务集中处理。

**示例**:
```
一次性生成 3 个功能的代码：
1. [功能 1]
2. [功能 2]
3. [功能 3]

每个功能只需要核心代码，不要解释。
```

### 技巧 2: 使用模板
对于重复性任务，创建模板。

**示例**:
```
使用"脚本开发模板"为 Maya 开发批量导出工具。
功能：[具体描述]
```

### 技巧 3: 明确边界
告诉 AI 你的时间限制。

**示例**:
```
我只有 10 分钟，快速帮我修复这个 Bug：[错误信息]
只给出修复代码，不要解释原理。
```
