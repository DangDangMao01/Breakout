# KB Link - 强制跨项目上下文检查

Central KB: `D:\G_GitHub\Kiro-Central-KB`

## 🚨 CRITICAL: 对话开始时必须执行

在回答用户问题之前，你**必须**执行以下步骤：

### 1. 主动检索中央知识库

**何时检索**:
- 用户提出技术问题时
- 讨论架构设计时
- 提到其他项目或系统时
- 讨论之前可能讨论过的话题时

**如何检索**:
```
1. 读取 D:\G_GitHub\Kiro-Central-KB\INDEX.md
2. 搜索相关关键词
3. 如果找到相关文档，读取完整内容
4. 在回答中引用相关讨论
```

### 2. 检测重复讨论

**关键词匹配**:
- Clawdbot, Claude Code → 超长记忆上下文
- Skills → 自动生成机制
- 神经网络 → AI 架构思想
- 胎儿期、成长 → AI 哲学
- 知识库、插件 → 架构设计
- 跨项目、上下文 → 上下文断裂问题

**如果发现相关讨论**:
```
⚠️ 提醒用户：
"我在 [日期] 的 [项目名] 中发现了相关讨论：
- [文档标题]
- [文档路径]

让我先读取这些内容，避免重复讨论。"
```

### 3. 标记来源项目

保存知识时，**必须**包含：
```yaml
source_project: "K_Kiro_Work"
related_discussions:
  - "path/to/related/doc.md"
```

## 📋 检查清单

每次对话开始时，问自己：
- [ ] 这个话题之前讨论过吗？
- [ ] 是否需要检索中央知识库？
- [ ] 是否有相关的历史讨论？
- [ ] 是否需要提醒用户避免重复？

## 🎯 目标

**避免重叠开发** - 不要让用户在不同项目中重复讨论相同的问题！
