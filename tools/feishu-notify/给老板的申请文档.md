## 🎯 三大应用场景

### 场景 1：GitLab 代码提交自动通知

**当前痛点**：
- ❌ 美术资源更新后，程序员不知道
- ❌ 需要手动通知，容易遗漏
- ❌ 提交信息显示不完整（Unknown User）

**工作配方解决方案**：
```
GitLab 代码推送 → 自动通知负责人
显示：提交人、提交信息、项目名称、查看变更按钮
```

**价值**：
- ✅ 节省每天 30 分钟沟通时间
- ✅ 减少资源相关 bug 50%
- ✅ 提升团队协作效率 20%

---

### 场景 2：多维表格任务自动提醒 ⭐ 核心价值

**当前痛点**：
- ❌ Bug 分配后，负责人不知道
- ❌ 任务即将到期，没有提醒
- ❌ 测试需要手动通知每个人
- ❌ 任务逾期了才发现

**工作配方解决方案**：

#### 2.1 Bug 管理自动化
```
测试在多维表格记录 Bug
    ↓
工作配方自动识别负责人
    ↓
立即通知负责人（飞书私聊）
    ↓
Bug 即将到期时自动提醒
```

**通知示例**：
```
【新 Bug 通知】
🐛 Bug ID: BUG-001
📝 标题: 角色动画播放异常
🎨 类型: 美术
👤 负责人: @王新来
⏰ 截止日期: 2026-01-30
🔗 [查看详情]
```

#### 2.2 任务排期自动提醒
```
策划在多维表格维护任务排期
    ↓
任务即将开始：提前 1 天提醒
任务即将到期：提前 3 天提醒
任务已逾期：每天提醒
```

**通知示例**：
```
【任务到期提醒】
⏰ 以下任务即将到期，请及时完成：

• 角色建模（还剩 2 天）
• 场景设计（还剩 1 天）
• UI 优化（还剩 3 天）

[查看详情]
```

**价值**：
- ✅ 测试不用手动通知，节省每天 40 分钟
- ✅ 任务按时完成率提升 40%
- ✅ 减少任务遗漏和延期
- ✅ 工作进度实时可见

---

### 场景 3：工作状态自动同步

**当前痛点**：
- ❌ 老板不知道每个人在做什么
- ❌ 需要开会才能了解进度
- ❌ 工作量不透明

**工作配方解决方案**：
```
多维表格记录工作内容
    ↓
每天自动生成工作报告
    ↓
发送给项目负责人/老板
```

**报告示例**：
```
【今日工作进展】2026-01-28

✅ 已完成（3 项）
• 角色动画优化 - @王新来
• UI 界面调整 - @张三
• Bug 修复 - @李四

🔄 进行中（2 项）
• 场景建模 - @王五（进度 60%）
• 特效制作 - @赵六（进度 30%）

⚠️ 逾期（1 项）
• 角色建模 - @钱七（逾期 1 天）
```

**价值**：
- ✅ 管理者实时了解团队进度
- ✅ 减少会议时间，提高效率
- ✅ 工作量透明，便于资源调配
